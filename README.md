# LabRecon

Research intelligence for UT Austin undergraduates. Search faculty research labs, explore publications and grants, and generate personalized cold outreach emails with AI.

## What it does

- **Search** — Find labs by PI name, research area, technique, or department. Results are relevance-scored and filterable by activity level and skills.
- **Lab profiles** — Each profile shows the PI's research summary, recent publications, active grants, and skill tags.
- **Outreach email generator** — Enter your profile once (name, major, year, skills, interests) and get a personalized cold email drafted by Claude, referencing the lab's most recent publication.
- **Tracker** — Save labs and track your outreach progress through five statuses: Saved → Sent → Responded → No Response → Meeting.

## Getting started

**Prerequisites**: Node.js 18+, an Anthropic API key.

```bash
# Install dependencies
npm install

# Seed the database with 15 UT Austin labs
npx tsx src/lib/db/seed.ts

# Add your API key
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env.local

# Start the dev server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Data

The database is seeded with 15 real UT Austin research labs across Computer Science, Biology, Neuroscience, Biomedical Engineering, and Physics. Each lab includes:

- Faculty name, title, department, and lab website
- AI-generated research summary
- Up to 3 recent publications (with venue, year, and abstract)
- Active grants with funder and amount
- Skill tags for filtering

All tracking data is stored in `localStorage` — no user accounts or server-side state.

## Email generation

The `/api/generate-email` route sends the lab's research summary and most recent publication to Claude with a carefully engineered prompt. The result is a sub-200-word cold email that:

- Opens with a specific finding from the referenced paper (not generic praise)
- Connects the student's skills to the lab's work
- Ends with a concrete ask for a 20-minute meeting

The student profile (name, major, year, skills, interests) is saved to `localStorage` and pre-fills on subsequent opens.
