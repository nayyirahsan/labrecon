# LabRecon — Impeccable Design Context

## Design Context

### Users
Juniors and seniors at UT Austin actively trying to break into faculty research labs — before grad school application deadlines, summer REU applications, or thesis enrollment windows. They open this tool with a job to do, not to browse. Session context: laptop, late at night, often stressed about timing, cross-referencing multiple tabs. They are smart, ambitious, and skeptical of anything that feels like it's dumbing things down for them.

### Brand Personality
**Three words: precise · driven · unimpressed**

LabRecon is a research instrument, not a companion app. It treats the user as a capable adult who wants facts, not encouragement. The interface should feel like something built for people who already know what they're doing — a tool that gets out of the way and lets the data speak.

### Aesthetic Direction

**Theme: Dark — always.** Students use this at 11 PM the week before an application deadline. Dark is purposeful, not affectation.

**Reference: Bloomberg Terminal × Linear**
- Bloomberg Terminal: maximum information density, no wasted space, functional typography, borders not shadows, data hierarchy through type weight and size not color saturation
- Linear: tight grid, precise spacing, keyboard-first interactions, subtle hover reveals, no marketing copy in the app shell

**Anti-references (what this must never look like):**
- Generic ed-tech: rounded pastel cards, emoji icons, "🚀 Start your research journey!" copy
- University websites: light blue brand colors, serif wordmarks, hero photos of students on grass
- AI startup aesthetic: purple-to-blue gradients, glowing orbs, "powered by AI" badges everywhere
- SaaS dashboard cliché: hero metric layout (big number + sparkline), sidebar with 12 nav items, card grid with identical icon+heading+text blocks

**Differentiation:** LabRecon should feel like something a grad student built for themselves and then open-sourced — opinionated, functional, slightly austere. The UI credibility mirrors the academic credibility of the content.

### Typography

**Display/Headings: Barlow Condensed** (weight 500–700)
- Condensed grotesque: tight, editorial, data-dense
- Reads as Bloomberg Terminal header at small sizes, editorial headline at large sizes
- Zero decoration — the condensed width itself creates hierarchy without needing extra visual treatment
- Replace the current Instrument Serif (on the reflex-reject list) with this

**Body/UI: Geist** (weight 400–500, already installed)
- Clean, technical, legible at 13–14px
- Correct for a dense information tool — doesn't romanticize the data

**Monospace: Geist Mono** (already installed)
- Citation counts, years, IDs, status codes

**Type scale (fixed rem, not fluid — this is an app UI, not a marketing page):**
- Display: 2rem / 500 / Barlow Condensed (section headers, lab names)
- Heading: 1.125rem / 600 / Barlow Condensed (card titles, professor names)
- Label: 0.6875rem / 500 / Geist, uppercase, 0.08em tracking (category tags, column headers)
- Body: 0.875rem / 400 / Geist (bios, descriptions)
- Caption: 0.75rem / 400 / Geist (citation counts, dates, secondary metadata)
- Mono: 0.75rem / 400 / Geist Mono (IDs, years)

**Line-height:**
- Display/headings: 1.05–1.1 (tight, Bloomberg-like)
- Body: 1.6 (dark bg needs slightly more breathing room than light)
- Labels: 1.0 (single line, no wrapping)

**Max line length:** 72ch for body text, no cap on table/data columns

### Color Palette

All in OKLCH. Base palette is zinc-950 with slate-900 cards, blue-500 primary.

--surface-elevated:  oklch(0.17  0.010  258)      /* hover state, active rows */
--border:            oklch(1  0  0 / 8%)          /* hairline borders */
--border-strong:     oklch(1  0  0 / 16%)         /* table dividers, separators */

--foreground:        oklch(0.97  0.002  264.5)    /* near-white, slightly cool */
--muted:             oklch(0.55  0.010  264.5)    /* secondary labels, metadata */
--placeholder:       oklch(0.40  0.006  264.5)    /* input placeholder, empty */

--primary:           oklch(0.623 0.214  259.1)    /* blue-500 — CTAs, links, active */
--primary-dim:       oklch(0.623 0.214  259.1 / 15%)  /* hover backgrounds */

--success:           oklch(0.696 0.170  162)      /* emerald-500 — "replied" status */

Accent usage rule: blue is used only for interactive elements (links, active states, buttons) and never as decoration. Emerald/amber/red are status colors only — they appear in badges, nowhere else.

### Spacing Scale

4pt base, semantic names:
- `--space-1`: 4px
- `--space-2`: 8px
- `--space-3`: 12px
- `--space-4`: 16px
- `--space-6`: 24px
- `--space-8`: 32px
- `--space-12`: 48px
- `--space-16`: 64px

Card padding: 20px (between space-4 and space-6 — slightly unusual, creates density)
Section gaps: 32–48px
Row height: 40px minimum for interactive rows

### Layout Philosophy

**Borders, not shadows.** This is a terminal-style UI. Use `1px solid var(--border)` everywhere, no box-shadows except for dropdown/popover elevation.

**Left-aligned, not centered.** Data tools align left. Only headings on empty states or zero-results screens get centered treatment.

**Column-based information layout.** Main content area uses a two-column split on desktop: a narrow left panel (280px) for filters/nav, wide right column for results. No sidebar with 12 items — max 6 nav items visible.

**Tables over cards for lists.** Professor search results render as a dense table (like Linear's issue list), not a card grid. Cards are for single-item detail views only.

**Progressive disclosure.** Default view shows name + department + 2 research areas + citation count. Full bio, publication list, email drafting tool appear in a slide-over Sheet or expandable row — not a separate page navigation.

### Motion

Minimal. This is a research tool, not a product demo.
- Row hover: `background-color` transition, 80ms ease-out
- Sheet open/close: `transform` slide, 200ms ease-out-quart
- Status badge change: `opacity` crossfade, 120ms
- Skeleton loaders: `opacity` pulse only, no shimmer
- No page transition animations — instant is faster

### Interaction Patterns

- **Search is primary.** The search bar is always visible and always focused on page load. No landing page hero — go straight to the tool.
- **Keyboard shortcuts.** Cmd+K for search, Cmd+/ for help panel, J/K for row navigation (like Linear, Notion).
- **Optimistic UI.** Tracking status changes update immediately, sync in background.
- **Empty states.** "No results for 'reinforcement learning'" + a suggested broader search term. Not a friendly illustration, not a CTA to "explore all labs."
- **Email draft state.** The AI draft generates inline in a Sheet panel, not a modal. User can edit in a textarea before copying.

### Design Principles

1. **Data first.** Every screen decision starts with: what information does the user need right now? Decoration is evidence against this principle.
2. **Density is a feature, not a flaw.** A Bloomberg terminal user doesn't want more whitespace — they want more visible data. Target 40px row height for results, not 72px cards.
3. **Professional credibility through restraint.** This tool helps students approach professors. If it looks like ed-tech, it undermines the user's credibility. Austerity is the brand.
4. **Status is sacred.** The tracker (interested / emailed / replied / rejected) is the most important UI element after search. Its four states need instant recognition — use the success/warning/danger palette nowhere else.
5. **Never repeat what the user can already see.** No section intros that restate the heading. No "Here are your results:" labels. No loading spinners that say "Loading your results…" — show a skeleton.

### shadcn/ui Customizations

- `--radius`: reduce to `0.25rem` (less rounded = more terminal/editorial)
- Button default variant: outlined, not filled — filled is for primary CTAs only
- Badge: all-caps, 0.6875rem, monospace for status badges (interested/emailed/replied/rejected)
- Table: zero padding on rows until hover, dense 40px row height
- Sheet: slides from right, 480px width on desktop (detail panel, email draft)
- Card: 1px border, no shadow, 20px padding
